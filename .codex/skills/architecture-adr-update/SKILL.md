# Architecture ADR Update

## Workflow
1. Identify decision scope and affected components.
2. Compare alternatives with concise pros/cons.
3. Record the selected option and consequences.
4. Save a short ADR entry using the template asset.

## Asset
- Use `assets/adr-template.md` as the starting format.

## Guardrails
- Keep ADR decision-focused, not implementation diary.
- Include migration impact and rollback considerations.
- Link touched files for traceability.
